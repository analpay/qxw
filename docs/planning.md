## 技术栈

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| CLI 框架 | Click | 命令行参数解析 |
| TUI 框架 | Textual | 终端用户界面 |
| 数据校验 | Pydantic v2 | 强类型约束 |
| ORM | SQLAlchemy 2.0 | 数据库映射 |
| 数据库 | SQLite | 轻量级本地存储 |
| 代码检查 | ruff | 代码格式化和 lint |
| 类型检查 | mypy | 静态类型分析 |

## 架构原则

1. **命令独立**: 每个命令是独立的入口，可单独使用
2. **领域驱动**: library/domain 下分 command/phrase/step 三层
3. **强类型**: 所有数据结构使用 Pydantic 约束
4. **统一错误处理**: 自定义异常体系，用户友好的错误提示
5. **TUI 优先**: 默认提供 TUI 交互界面，支持 `--no-tui` 降级
